Potato Potato?

[Ruby Hashes vs. JavaScript Objects]

December 22, 2015

In Ruby, hashes are used to store data in key-value pairs. In JavaScript, objects store data in pairs also, and they are typically described as name-value or property-value pairs. In this article I will look at the similarities and differences between the two structures. From here on, 'objects' will refer to JavaScript objects, even though Ruby hashes are of course also objects in Ruby.

Creating a new empty hash is very similar to creating a new empty object.

//JavaScript
var obj = new Object();
var obj2 = {};

#Ruby
hash = Hash.new
hash2 = {}

The syntax differences widen as we start to add value pairs into each one. First, let's look at initializing a hash and object with pairs already:

//JavaScript
var obj3 = {
  firstName: "Helen",
  lastName: "Smith",
  age: 30
  spouse: {
    firstName: "Joe",
    lastName: "Smith",
    age: 30
  }
};

#Ruby
hash3 = {
  "first_name" => "Helen",
  :last_name => "Smith",
  age: 30,
  spouse: {
    first_name: "Joe",
    last_name: "Smith",
    age: 30
  }
}

Alright! The above should convey that the essence of the two structures is almost identical. Syntactically though, there are several differences. There are multiple ways to write Ruby key-value pairs: in the above instance, one key is written as a string, the next as a symbol with a leading colon, and the next as a symbol with a trailing colon. The => is used at times and not at others (though it would never be proper convention to mix and match within the same hash or even the same program). Meanwhile, in the JavaScript above, there is really only one way to write out the property, with the colon after the property name. Another important feature for both structures is the ability to nest other types of data structures inside. Above, a hash is nested inside of a hash, and a JavaScript object is nested inside of the original object.

In Ruby, any object can be a key or value. So a key could be an array, hash, integer, string, or any other Ruby object. In a JavaScript object, the properties must be converted to strings. Consider the following:

//JavaScript
var array = [1, 2, 3];
var obj = {
  array: "array",
  45: "number"
};
for (var key in obj)
  console.log (typeof key)

>> string
>> string

Above, both the array and number are converted to strings when they are added as properties.

Adding values to existing hashes or objects is similar, though JavaScript commonly uses dot notation when the property name is a valid variable name, while Ruby uses bracket notation. JavaScript does allow bracket notation as well, which is useful for when the string property name has a space or is a number, for example.

//JavaScript
obj3.middleName = "Lisa";
obj3["favoriteColor"] = "blue";

#Ruby
hash3["middle_name"] = "Lisa"
hash3["favorite_color"] = "blue"

Last but not least, Ruby hashes and JavaScript objects each have their own built-in methods and they do differ. The Ruby Docs will have all the hash methods, and the Mozilla Developer Network has the object functions.

There are a few similar methods/functions, such as the Ruby has_key? method and the JavaScript hasOwnProperty function, but there are many different ones, so I urge you to check the documentation out!

Hope this has been a helpful comparison of Ruby hashes and JavaScript objects. Until next time...
