On The Map

[Introduction to the Ruby Enumerable#map Method]

December 10, 2015

The Enumerable module in Ruby allows collection objects to access a subset of behaviors and methods defined in terms of its each method. The each method will outline how an object iterates through its elements. Once the each method is implemented, the Enumerable module is able to provide a wide range on methods onto the collection object.

One very valuable method within the Enumerable module is map. Map differs from each because it returns a new array object, while each only returns its original object. Here is how each returns a code block:

array = [1, 2, 3]
>> [1, 2, 3]
array.each {|x| x * 10 }
>> [1, 2, 3]
result = array.each {|x| x * 10 }
>> [1, 2, 3]
print result
[1, 2, 3]
>> nil

Each does not save the values resulting from running its code block; they are discarded, and the only thing left to return is the original receiver object. Meanwhile map's return value is much more clear:

array = [1, 2, 3]
>> [1, 2, 3]
array.map {|x| x * 10 }
>> [10, 20, 30]

From there, the new array can be used however it wants.
