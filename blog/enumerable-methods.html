On The Map

[Introduction to the Ruby Enumerable#map Method]

December 10, 2015

The Enumerable module in Ruby allows a collection object to access a subset of behaviors and methods defined in terms of its each method. The each method will outline how an object iterates through its elements. Once the each method is implemented, the Enumerable module is able to provide a wide range of methods onto the collection object.

One very valuable method within the Enumerable module is map. Map differs from each because it returns a new array object, while each only returns its original object. Here is how each returns a code block:

array = [1, 2, 3]
>> [1, 2, 3]
array.each {|x| x * 10 }
>> [1, 2, 3]
result = array.each {|x| x * 10 }
>> [1, 2, 3]
print result
[1, 2, 3]
>> nil

Each does not save the values resulting from running its code block; they are discarded, and the only thing left to return is the original receiver object. Meanwhile map's return value is much more clear:

array = [1, 2, 3]
>> [1, 2, 3]
array.map {|x| x * 10 }
>> [10, 20, 30]
result = array.map {|x| x * 10 }
>> [10, 20, 30]
p result
[10, 20, 30]
>> [10, 20, 30]

From there, the new array can be used however it wants. So map returns a remapping of the original object into a new transformed array. Meanwhile, the original object remains unchanged:

p array
[1, 2, 3]
>> [1, 2, 3]

To change the original array, you can use map!, the destructive form of map.Map! transforms the original object in place.

There are other useful ways to use map. Combining it with other methods is easy. For example:

array = ["mary", "helen", "john", "william"]
>> ["mary", "helen", "john", "william"]
array.map(&:capitalize)
>> ["Mary", "Helen", "John", "William"]

Without going too far into blocks and Procs in Ruby, map(&:method_name) is an idiomatic way of mapping a method to an object. The above example would be functionally equivalent to array.map {|word| word.capitalize }, though in fact the method in the shorthand version is being expressed as a Proc and converted into a block. However, that riveting discussion shall be saved for another blog post!

Hopefully this has been a helpful introduction to the value of the map method and its primary difference versus each. Importantly, the collect method is the exact same method as map by a different name, so if you see it, you should now know what its doing! Hasta pasta amigos!
